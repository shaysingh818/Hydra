Summary

Hydra is a system for implementing reinforcement learning algorithms. This paradigm of learning requires us to represent an agent, state and environment. Games, specifically with two dimensional environments can easily represent RL problems. Algorithms can be used to manage the state and create learning strategies. The purpose of this project, is to learn about various methods to represent a game environment, create libraries for implementing reinforcement learning algorithms and understanding the fundamentals of creating “simple” games.

Machine Learning Strategy

This purpose of the engine is to research different machine learning methods for developing AI’s that learn to solve puzzles easily. The game’s are not the same, as a result, it will require different ML models and environment representations. For each game, this will be the format for developing algorithms and environments.

  1. Represent the environment of the game
  1. Test the environment, make sure it’s playable
  1. Research RL/ML algorithms to learn the environment
  1. Create bench marking and test cases
  1. Test accuracy of implemented models

This section outlines the requirements needed to represent these games as two dimensional environments. While most of the environments have conventional rules, we can customize the environments and add options to make it easier or harder for an agent to learn in the environment.

Connect 4

These are the requirements to represent the game environment of Connect 4. This won’t include the GUI and how the environment will be rendered to the user.

Connect 4: https://www.fgbradleys.com/rules/Connect Four.pdf

User interface: Create UI concept for the connect 4 board.

  1. The board is represented as two dimensional grid. 6 rows by 7 columns. Each slot should be accessible to place a piece
  1. The player must select which column to drop a piece. There should be logic to slide a piece to the lowest available row.
  1. The pieces for the game are made up of 21 black checkers and 21 red checkers. The game engine should be able to abstractly represent the pieces.
  1. There should be game cycle that alternates between players. Player 1 will represent the black checkers and player 2 should represent the red checkers.
  1. Need a function that can get all available positions in a connect 4 environment.
  1. A player can win if they get 4 in a row vertically, horizontally or diagonally.

Game Of Life

The Game of Life, also known simply as Life, is a cellular automaton devised by the British mathematician John Horton Conway in 1970. It is a zero-player game, meaning that its evolution is determined by its initial state, requiring no further input. These are the rules for the game of life.

  1. Any live cell with fewer than two live neighbors die, by under population
  1. Any live cell with tow or three live neighbors lives on to the next generation
  1. Any live cell with more than three live neighbors die, by overpopulation
  1. Any dead cell with three live neighbors becomes a live cell, by reproduction

The rules can be condensed into

  1. Any live cell with two or three neighbors survives
  1. Any dead cell with three live neighbors becomes a live cell
  1. All other live cells die in the next generation, all other dead cells stay dead.

Overall, this game will be used to explore concepts in cellular automata and use models to evaluate evolutionary algorithms. Use the genetic algorithms book to design models for how we could make this simulation more intriguing.
